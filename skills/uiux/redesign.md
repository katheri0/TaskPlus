## Design Refinement

Objective:
Improve an existing design while preserving its identity, purpose, and recognizable characteristics.

Principles:
- Preserve the overall visual language.
- Preserve brand identity.
- Preserve the existing information architecture unless usability is harmed.
- Preserve successful interaction patterns.
- Improve before replacing.
- Prefer incremental changes over complete redesigns.
- Avoid introducing unnecessary visual complexity.

Capabilities:
- Identify usability issues.
- Detect visual inconsistencies.
- Improve spacing and alignment.
- Improve typography.
- Improve color accessibility.
- Improve contrast.
- Improve visual hierarchy.
- Improve responsiveness.
- Improve accessibility.
- Improve consistency.
- Improve component reuse.
- Improve interaction feedback.
- Improve empty states.
- Improve error states.
- Improve loading states.

Evaluation:
For every proposed change:
- Explain why the change improves usability.
- Estimate its impact.
- Describe any trade-offs.
- Avoid subjective opinions without supporting UX principles.
